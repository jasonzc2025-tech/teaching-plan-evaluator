0001: # 路线A Docker 部署说明
0002: 
0003: ## 适用场景
0004: - 继续使用阿里云 ECS
0005: - 不再依赖宝塔里的旧 Python 运行环境
0006: - 先保留 SQLite 与本地文件
0007: - 浏览器访问的 **Web 应用**；对外测试入口为 **8081**（经 Nginx），与旧站端口可并存
0008: 
0009: ## 部署前检查（上传服务器 / 打压缩包前）
0010: 1. 已复制 `.env.example` → `.env`，并填写：`SECRET_KEY`、`ADMIN_PASSWORD`、`LLM_API_KEY`（及按需的 `LLM_API_BASE`、`LLM_MODEL_NAME`）。
0011: 2. 生产环境保持 `EVAL_EXPOSE_INTERNAL_ERRORS=false`（勿对公网暴露内部异常）。
0012: 3. 若需提交软著源程序合订本，在仓库根目录执行：  
0013:    `python scripts/prepare_deploy.py`  
0014:    （会清理 `__pycache__`、`.pyc` 并重新生成 `artifacts/source_submission_full.txt`。）
0015: 4. 确认存在 `static/vendor/chart.umd.min.js`（后台图表用，不依赖外网 CDN）。
0016: 5. 安全组：建议仅放行 **8081**；`5001` 仅绑定本机时外网无法访问属正常。
0017: 
0018: ## 目录说明
0019: - `Dockerfile`：应用镜像构建文件
0020: - `docker-compose.yml`：一键启动 Web + Nginx
0021: - `nginx/default.conf`：反向代理配置
0022: - `.env.example`：环境变量模板
0023: - `instance/`：SQLite 数据库持久化目录
0024: - `data/uploads/`：预留上传文件持久化目录
0025: 
0026: ## 服务器准备
0027: ### Ubuntu/Debian
0028: 按 Docker 官方文档，推荐通过 Docker 的 apt 仓库安装 Docker Engine。 
0029: 
0030: ### CentOS Stream 9/10
0031: 按 Docker 官方文档，推荐通过 Docker 的 rpm 仓库安装 Docker Engine。 
0032: 
0033: ## 部署步骤
0034: 1. 把整个目录上传到 ECS，例如 `/opt/teaching-plan-evaluator-webapp`。
0035: 2. 复制 `.env.example` 为 `.env`，至少改这些值：
0036:    - `SECRET_KEY`
0037:    - `ADMIN_PASSWORD`
0038:    - `LLM_API_KEY`
0039:    - `LLM_API_BASE`
0040:    - `LLM_MODEL_NAME`
0041: 3. 在项目目录执行：
0042:    ```bash
0043:    docker compose up -d --build
0044:    ```
0045: 4. 访问测试地址（**外网请用 8081**，见下）：
0046:    - **推荐**：走 Nginx：`http://服务器IP:8081/`
0047:    - `docker-compose.yml` 中 Flask 映射为 `127.0.0.1:5001:5000`，表示 **5001 仅服务器本机可访问**（公网浏览器访问 `http://服务器IP:5001/` 会失败，属预期）。本机调试可在服务器上 `curl http://127.0.0.1:5001/health`，或通过 SSH 隧道把 5001 转到本机。
0048: 5. 后台地址：`/admin`
0049: 
0050: ## 推荐开放端口
0051: - **8081**：测试阶段对外访问（经 Nginx，建议安全组只开此端口）
0052: - **5001**：仅本机（或 SSH 隧道）访问 Flask；若确需公网直连 Flask，把 compose 中端口改为 `"5001:5000"` 并收紧安全组
0053: - 以后切正式站时，只保留 80/443 更合适
0054: 
0055: ## 常用命令
0056: ```bash
0057: # 启动
0058:  docker compose up -d --build
0059:

0063: 
0064: # 重启
0065:  docker compose restart
0066: 
0067: # 停止
0068:  docker compose down
0069: ```
0070: 
0071: ## 切换正式站的最稳方法
0072: 1. 先保持旧站继续跑
0073: 2. 新版在 8081（及对内的 5001）完整测试
0074: 3. 测完后再把域名或反向代理切到新版
0075: 4. 确认新版稳定后，再停旧站
0076: 
0077: ## 回滚
0078: 只要旧站没删，回滚就是把访问入口重新指回旧站。

0001: # 住培临床教案质量智能评审系统 V1.0
0002: 
0003: 这是一个基于 Flask 的 Web 应用，用于对住院医师规范化培训相关教案进行结构化评审、问题归类和研究型统计分析。
0004: 
0005: ## 核心能力
0006: 
0007: - 教案文本上传与粘贴评审
0008: - 五类教案自动识别或手动指定
0009: - 结构化输出：总分、条款分、问题列表、优点、修改建议
0010: - 研究型落库：问题分类、严重程度、条款失分、版本信息
0011: - 后台分析：类型分布、问题热度、严重程度、近14天趋势
0012: - CSV 导出：记录表、问题表、条款表
0013: - 软著辅助脚本：自动生成源程序交存版文本

0001: from pathlib import Path
0002: 
0003: 
0004: ROOT = Path(__file__).resolve().parents[1]
0005: OUTPUT_DIR = ROOT / "artifacts"
0006: OUTPUT_DIR.mkdir(exist_ok=True)
0007: 
0008: TARGET_SUFFIXES = {".py", ".html", ".css", ".md", ".txt"}
0009: EXCLUDE_PARTS = {".venv", "__pycache__", "instance", "artifacts"}
0010: 
0011: 
0012: def should_include(path: Path) -> bool:
0013:     return path.suffix in TARGET_SUFFIXES and not any(part in EXCLUDE_PARTS for part in path.parts)
0014: 
0015: 
0016: def main():
0017:     output_path = OUTPUT_DIR / "source_submission_full.txt"
0018:     with output_path.open("w", encoding="utf-8") as handle:
0019:         for path in sorted(ROOT.rglob("*")):
0020:             if path.is_file() and should_include(path):
0021:                 rel = path.relative_to(ROOT)
0022:                 handle.write("=" * 90 + "\n")
0023:                 handle.write(f"FILE: {rel}\n")
0024:                 handle.write("=" * 90 + "\n")
0025:                 with path.open("r", encoding="utf-8", errors="ignore") as src:
0026:                     for idx, line in enumerate(src, start=1):
0027:                         handle.write(f"{idx:04d}: {line}")
0028:                 handle.write("\n\n")
0029:     print(f"生成完成: {output_path}")
0030: 
0031: 
0032: if __name__ == "__main__":
0033:     main()

0001: """
0002: 部署前准备：清理 __pycache__ / .pyc，并重新生成软著交存文本。
0003: 在项目根目录执行: python scripts/prepare_deploy.py
0004: """
0005: from __future__ import annotations
0006: 
0007: import shutil
0008: import subprocess
0009: import sys
0010: from pathlib import Path
0011: 
0012: 
0013: ROOT = Path(__file__).resolve().parents[1]
0014: SKIP_DIR_NAMES = {".venv", ".git", "instance", "artifacts"}
0015: 
0016: 
0017: def remove_caches() -> tuple[int, int]:
0018:     removed_dirs = 0
0019:     removed_files = 0
0020:     for path in ROOT.rglob("*"):
0021:         try:
0022:             rel = path.relative_to(ROOT)
0023:         except ValueError:
0024:             continue
0025:         if any(p in SKIP_DIR_NAMES for p in rel.parts):
0026:             continue
0027:         if path.is_dir() and path.name == "__pycache__":
0028:             shutil.rmtree(path, ignore_errors=True)
0029:             removed_dirs += 1
0030:         elif path.is_file() and path.suffix == ".pyc":
0031:             path.unlink(missing_ok=True)
0032:             removed_files += 1
0033:     return removed_dirs, removed_files
0034: 
0035: 
0036: def main() -> int:
0037:     d, f = remove_caches()
0038:     print(f"已删除 __pycache__ 目录: {d}，.pyc 文件: {f}")
0039:     script = ROOT / "scripts" / "build_source_submission.py"
0040:     r = subprocess.run([sys.executable, str(script)], cwd=str(ROOT))
0041:     return r.returncode
0042: 
0043: 
0044: if __name__ == "__main__":
0045:     raise SystemExit(main())
