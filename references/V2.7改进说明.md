# 教案评审系统 V2.6 → V2.7 改进对比报告

**生成时间**: 2026年4月18日  
**文档版本**: V2.7改进说明  
**适用对象**: 临床教案评审智能体升级说明

---

## 一、改进总览

V2.7在V2.6基础上进行了**五大核心优化**，从评分精度、类型识别、证据权重、否决处理、目标评估五个维度全面提升评审质量。

| 改进维度 | V2.6 | V2.7 | 改进效果 |
|:---------|:-----|:-----|:---------|
| **评分尺度** | 四级判定 | 五级锚定 | 精度提升25% |
| **类型识别** | 人工+简单提示 | TCI量化指数 | 可量化、可验证 |
| **硬证据** | 统一三联规则 | 动态权重 | 五类教案差异化 |
| **否决处理** | 一票否决 | 降级缓冲带 | 更灵活、更人性化 |
| **目标评估** | 四条件检查 | 四维矩阵 | 深度、全面、可追踪 |

---

## 二、详细改进说明

### 2.1 评分尺度：四级 → 五级锚定 (P0)

#### V2.6问题
- 四级判定（优秀/合格/基本合格/不合格）粒度粗
- "合格"与"基本合格"边界模糊，评审员主观判断空间大
- 缺乏对"良好"等级的明确定义

#### V2.7改进
新增**五级评分锚定体系**：

| 等级 | 分数区间 | 定义 | 处理建议 |
|:-----|:--------:|:-----|:---------|
| **优秀** | 90-100 | 各条款均达优秀，无明显短板 | 示范教案，供学习参考 |
| **良好** | 80-89 | 整体质量高，个别条款可提升 | 针对性改进建议 |
| **中等** | 70-79 | 基本达标，多处可改进 | 明确改进清单，限期完善 |
| **及格** | 60-69 | 刚达最低标准，明显缺陷 | 必须修改，重新提交 |
| **不及格** | <60 | 未达基本质量标准 | 退回重写，系统学习后重提 |

**改进价值**: 评分更精准，反馈更有针对性，帮助教师明确改进方向。

---

### 2.2 类型一致性指数(TCI) (P1)

#### V2.6问题
- 类型识别依赖人工检查，效率低
- 首页标注与内容实质不一致的情况难以量化
- 缺乏客观指标评估类型匹配度

#### V2.7改进
新增**TCI量化评估模型**：

```
TCI = L1(20%) + L2(30%) + L3(50%)
```

| 层级 | 权重 | 检查内容 | 检测方法 |
|:-----|:----:|:---------|:---------|
| **L1首页标签** | 20% | 申报类型勾选 | 关键词提取 |
| **L2教学目标** | 30% | 类型特征词密度 | NLP语义分析 |
| **L3实施流程** | 50% | 标准结构匹配 | 流程模式识别 |

**五类教案特征词表**:
- **教学查房**: 示教室、床旁查房、病历汇报、体格检查、临床思维
- **病例讨论**: 诊断思路、鉴别诊断、诊疗决策、病例分析
- **临床小讲课**: 知识点、理论讲解、幻灯、授课、学习要点
- **技能培训**: 操作步骤、DOPS、模拟训练、技能考核、实操
- **教学门诊**: 独立接诊、SOAP病历、Mini-CEX、接诊评估

**改进价值**: 类型识别从"凭感觉"变为"可量化"，提供客观数据支持类型判定。

---

### 2.3 硬证据权重动态化 (P1)

#### V2.6问题
- 所有教案类型使用统一的硬证据三联规则
- 不同教案类型的关键证据未做区分
- 技能培训的"应急预案"与教学查房的"知情同意"同等权重，不合理

#### V2.7改进
新增**五类教案硬证据权重矩阵**：

| 硬证据条款 | 教学查房 | 病例讨论 | 临床小讲课 | 技能培训 | 教学门诊 |
|:-----------|:--------:|:--------:|:----------:|:--------:|:--------:|
| **知情同意** | 1.5× | - | - | - | 2.0× |
| **应急预案** | 1.0× | - | - | **2.0×** | 1.0× |
| **风险评估** | 1.2× | 1.0× | 0.8× | 1.5× | 1.2× |
| **操作流程** | - | - | 0.5× | **2.0×** | - |
| **DOPS匹配** | - | - | - | 1.5× | 1.0× |
| **案例分层** | 0.8× | 1.2× | **1.5×** | 1.2× | 0.8× |
| **病例选择** | 1.2× | **1.5×** | 0.8× | - | 1.2× |

**评分公式**:
```
硬证据得分 = 基础分 × 类型权重 × 证据完整度
```

**改进价值**: 不同类型教案的关键要素得到差异化重视，评分更符合临床实际。

---

### 2.4 否决项降级缓冲带 (P2)

#### V2.6问题
- 一票否决过于刚性，无整改空间
- 轻微瑕疵与严重缺陷同等处理
- 教师容易因小问题被全盘否定，挫败感强

#### V2.7改进
新增**三级降级缓冲带**：

| 级别 | 颜色 | 扣分 | 触发条件 | 处理方式 |
|:-----|:----:|:----:|:---------|:---------|
| **红色** | 🔴 | -40分 | 严重违反核心安全规范 | 必须整改后重新提交 |
| **橙色** | 🟠 | -15分 | 重要证据缺失或重大缺陷 | 限期整改，二次评审 |
| **黄色** | 🟡 | -5分 | 轻微瑕疵或细节不足 | 建议优化，可继续流程 |

**各类型缓冲带映射示例（教学查房）**:

| 否决项 | 红色(-40) | 橙色(-15) | 黄色(-5) |
|:-------|:----------|:----------|:---------|
| **知情同意** | 完全未提及；模板残留 | 仅写"已告知" | 有告知但不够具体 |
| **风险评估** | 完全未考虑患者隐私 | 仅有原则无措施 | 有措施但缺范围 |
| **核心指南** | >5年且作为核心依据 | >5年但作为次要参考 | 未标注年份 |

**否决保留条款**（仍维持一票否决）:
- 伪造病历或操作记录
- 严重违反医学伦理
- 核心安全规范完全缺失且可能造成伤害

**改进价值**: 从"一棍子打死"变为"分级处理"，既保证安全底线，又给予改进机会。

---

### 2.5 目标达成度矩阵 (P2)

#### V2.6问题
- 教学目标仅检查"四条件"（具体/可测/匹配/分级）
- 缺乏对目标质量的深度评估
- 无法提供针对性的目标优化建议

#### V2.7改进
新增**四维评估矩阵**：

| 维度 | 权重 | 评估要点 | 评分细则 |
|:-----|:----:|:---------|:---------|
| **完整性** | 25% | 目标数量2-4个，覆盖知识/技能/态度 | 20-25分: 数量适中+覆盖≥2维度 |
| **合理性** | 25% | 难度与对象匹配，与课时匹配 | 20-25分: 难度分层清晰 |
| **可测性** | 25% | 使用行为动词，量化阈值明确 | 20-25分: 全部行为动词+量化 |
| **匹配度** | 25% | 与类型匹配，与后文闭环 | 20-25分: 类型匹配+≥2处闭环 |

**输出格式**:
```json
{
  "目标达成度矩阵": {
    "total_score": 82,
    "completeness": { "score": 20, "target_count": 3, "issues": [] },
    "reasonableness": { "score": 22, "difficulty_layers": 3, "issues": [] },
    "measurability": { "score": 18, "action_verbs": [...], "issues": [...] },
    "alignment": { "score": 22, "type_match": "匹配", "closure_points": 3, "issues": [] },
    "improvement_suggestions": ["为第2目标增加量化阈值"]
  }
}
```

**改进价值**: 从"有没有"变为"好不好"，提供具体、可操作的目标优化建议。

## 三、V2.7评分流程

```
【阶段一：预检层】
  ├─ 首页规范性检查
  ├─ 附件与正文对应关系
  ├─ 模板残留识别
  ├─ TCI计算（类型一致性指数） ← V2.7新增
  └─ 提示词注入检测

【阶段二：通用维度评分】
  ├─ 首页规范性（4分）
  ├─ 教学目标（10分）→ 同步输出目标达成度矩阵 ← V2.7新增
  ├─ 病例选择（10分）
  ├─ 教学设计（11分）
  ├─ 参考资料（5分）
  └─ 风险评估（5分）→ 应用硬证据动态权重 ← V2.7新增

【阶段三：分项维度评分】（按申报类型）
  ├─ 按类型选择对应条款表
  ├─ 应用硬证据权重动态化 ← V2.7新增
  └─ 检查否决项缓冲带级别 ← V2.7新增

【阶段四：汇总与判定】
  ├─ 计算 raw_total
  ├─ 应用否决项降级缓冲带 ← V2.7新增
  ├─ 计算 adjusted_score
  ├─ 五级锚定等级判定 ← V2.7改进
  ├─ 触发异常分复核
  └─ 输出完整JSON与校验日志
```

## 五、改进效果预期

### 量化提升

| 指标 | V2.6 | V2.7预期 | 提升幅度 |
|:-----|:-----|:---------|:---------|
| 评分精度 | 4级 | 5级 | +25% |
| 类型识别 | 定性 | TCI量化 | 可量化 |
| 问题发现 | 11处/10份教案 | 16处/10份教案 | +45% |
| 反馈针对性 | 一般 | 高（具体改进清单） | 显著提升 |
| 评审一致性 | 依赖评审员 | 算法辅助标准化 | 降低主观偏差 |

### 质性提升

1. **更精准**: 五级锚定避免"合格"与"基本合格"的模糊地带
2. **更客观**: TCI指数提供类型匹配度的量化依据
3. **更合理**: 动态权重体现不同教案类型的关键差异
4. **更人性化**: 缓冲带给予教师整改机会，降低挫败感
5. **更深入**: 四维矩阵全面评估教学目标质量

## 七、使用建议

### 对评审员
1. 优先查看TCI指数，快速判断类型是否匹配
2. 关注缓冲带级别，区分轻微瑕疵与严重问题
3. 参考目标达成度矩阵，提供具体改进建议

### 对教案作者
1. 申报前自查TCI，确保类型选择正确
2. 关注硬证据权重高的条款（如技能培训的应急预案）
3. 根据四维矩阵优化教学目标

### 对管理者
1. 使用五级锚定进行教案分级管理
2. 利用TCI数据监控类型申报准确率
3. 基于缓冲带数据识别培训需求
