# 版本历史

## V3.0（2026年4月）

### 新增
- 流式实时输出（SSE）：LLM token 逐步推送至前端，评审过程可视化
- UI 全面重设计：深蓝渐变顶栏、评分大字 KPI、结论横幅颜色区分
- 自动元数据解析：从文件名识别科室、带教老师、课程名称，无需手填
- 后台带教老师/科室多维统计
- 多轮评审追踪：同一课程多次提交自动记录轮次、首次分、最高分
- 最近记录表新增科室、老师、课程、评审轮次列
- gunicorn 改用 gevent 异步 worker

### 修复
- 修复 PDF 解析缺少 poppler-utils
- 修复 review_round 非数字输入导致 500
- 修复恶意表名导出返回 500 改为 404
- 修复图表比例异常，Chart.js 改用国内 CDN
- 修复 zipfile 文件句柄泄露

### 安全
- SECRET_KEY 未配置时生成随机密钥并警告
- Session Cookie 加固 HttpOnly/SameSite
- Flask 直连端口绑定至 127.0.0.1
- .env 加入 .dockerignore

## V2.6（2026年3月）

### 主要特性
- 百分制条款评分体系（通用维度45分 + 分项维度55分）
- 五类教案完整支持
- 三联证据评分机制
- 否决项零容忍规则
- 评审防操纵机制
- 模块化提示词架构（prompt_blocks 目录）
- 研究型后台与 CSV 导出
- Claude Skill 文件发布

### 应用成效（截至2026年4月）
- 累计完成教案评审 450 余份
- 服务带教医师百余人次
- 单份评审时间从数十分钟缩短至3分钟以内
- 单份评审成本约 0.02 元
- 已在吉林医药学院附属医院、上海市第六人民医院、蚌埠医科大学第一临床医学院等多家跨区域机构测试验证

## V1.0（2026年1月）

- 项目启动，完成核心评审规则设计与提示词体系构建
- 以 Skill 形式发布，支持在主流大语言模型平台加载使用
- 实现五类教案基础评审能力，输出结构化评审意见
- 完成跨平台兼容性验证
